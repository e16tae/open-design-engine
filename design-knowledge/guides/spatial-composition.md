# 공간 구성 (Spatial Composition) 디자인 가이드

이 가이드는 디자인 에이전트가 공간을 효과적으로 구성하기 위한 지침을 제공합니다.
간격(spacing), 정렬(alignment), 밀도(density), 시각적 리듬(visual rhythm)에 대한
구체적인 수치와 규칙을 포함합니다.


## 핵심 원칙

### 1. 게슈탈트 원리 (Gestalt Principles)

인간의 시각 인지를 기반으로 한 공간 구성 원리입니다.

**근접성 (Proximity)**
서로 가까운 요소는 관련된 것으로 인식됩니다. 관련 요소 사이의 간격은
관련 없는 요소 사이의 간격보다 작아야 합니다.

- 관련 요소 간격: 8-16px (1x-2x base)
- 섹션 간 간격: 24-48px (3x-6x base)
- 그룹 간격은 내부 간격의 2배 이상이어야 합니다.

**유사성 (Similarity)**
시각적으로 유사한 요소(크기, 색상, 형태)는 같은 그룹으로 인식됩니다.
일관된 간격 시스템은 유사성을 강화합니다.

**연속성 (Continuity)**
정렬된 요소는 연결된 것으로 인식됩니다. 그리드 정렬이 중요한 이유입니다.

**폐합 (Closure)**
불완전한 형태도 완전한 것으로 인식하려는 경향. 카드 레이아웃,
그룹핑 보더 등에서 활용됩니다.

### 2. 시각적 계층 (Visual Hierarchy)

공간을 통해 정보의 중요도를 전달합니다.

- **크기(Size):** 중요한 요소일수록 더 큰 공간을 차지합니다.
- **간격(Spacing):** 중요한 요소 주위에 더 많은 여백을 둡니다.
- **위치(Position):** 중요한 정보를 시각적 진입점(상단-좌측 또는 중앙)에 배치합니다.
- **밀도(Density):** 핵심 콘텐츠 영역은 적절한 밀도를, 보조 영역은 느슨한 밀도를 유지합니다.

### 3. 의도적 여백 (Intentional Whitespace)

여백은 빈 공간이 아니라 적극적인 디자인 요소입니다.

- **매크로 여백 (Macro whitespace):** 섹션 간, 컬럼 간 큰 여백
  - 시각적 휴식을 제공하고 콘텐츠 구조를 전달합니다.
  - 최소 24px, 권장 32-64px
- **마이크로 여백 (Micro whitespace):** 요소 내부, 줄 사이 작은 여백
  - 가독성을 향상시키고 시각적 호흡을 제공합니다.
  - 텍스트 줄 간격, 아이콘과 레이블 사이 간격 등
- 모든 여백은 의도를 가져야 합니다: "이 간격이 왜 이 값인가?"


## 규칙

### 1. 간격 스케일 (Spacing Scale)

모든 간격은 기본 단위(base unit)의 배수여야 합니다.

**4px Base Grid (마이크로 스케일)**

| 배수 | 값 | 용도 |
|---|---|---|
| 1x | 4px | 아이콘 내부 간격, 매우 밀접한 요소 |
| 2x | 8px | 관련 요소 간 기본 간격, 인라인 패딩 |
| 3x | 12px | 리스트 아이템 간격, 소형 패딩 |
| 4x | 16px | 카드 내부 패딩, 폼 요소 간격 |
| 5x | 20px | 중간 패딩 |
| 6x | 24px | 섹션 내부 패딩, 컨테이너 패딩 |
| 8x | 32px | 섹션 간 간격 |
| 10x | 40px | 큰 섹션 간격 |
| 12x | 48px | 페이지 마진, 대형 섹션 간격 |
| 16x | 64px | 히어로 섹션 패딩 |

**8px Layout Grid (매크로 스케일)**

레이아웃 수준의 간격은 8px 배수를 사용합니다:

| 배수 | 값 | 용도 |
|---|---|---|
| 1x | 8px | 기본 그리드 단위 |
| 2x | 16px | 컬럼 거터 (모바일) |
| 3x | 24px | 컬럼 거터 (데스크톱) |
| 4x | 32px | 섹션 마진 |
| 6x | 48px | 콘텐츠 패딩 (모바일) |
| 8x | 64px | 콘텐츠 패딩 (데스크톱) |
| 12x | 96px | 페이지 사이드 마진 |
| 16x | 128px | 히어로/랜딩 여백 |

### 2. 근접성 규칙 (Proximity Rules)

관련 요소와 비관련 요소의 간격 비율:

```
그룹 내부 간격 : 그룹 간 간격 = 1 : 2 이상
```

**구체적 가이드라인:**

- 폼 필드 레이블과 인풋: 4-8px
- 폼 필드 간 간격: 16-24px
- 폼 그룹 간 간격: 32-48px
- 카드 내부 요소 간격: 8-16px
- 카드 간 간격: 16-24px
- 내비게이션 아이템 간격: 8-16px (수평), 4-8px (수직)
- 섹션 헤딩과 콘텐츠: 8-16px
- 섹션 간 간격: 32-64px

### 3. 정렬 (Alignment)

모든 요소는 그리드 또는 다른 요소에 정렬되어야 합니다.

**정렬 유형:**
- **그리드 정렬:** 컬럼 그리드에 맞춤 (8px 단위)
- **엣지 정렬:** 인접 요소의 가장자리에 정렬
- **센터 정렬:** 컨테이너 또는 다른 요소의 중심에 정렬
- **베이스라인 정렬:** 텍스트의 베이스라인에 정렬

**정렬 우선순위:**
1. 좌측 정렬 (LTR 언어 기본)
2. 우측 정렬 (RTL 언어 기본, 숫자 데이터)
3. 중앙 정렬 (헤딩, 히어로 섹션)
4. 양쪽 정렬 (본문 텍스트, 주의하여 사용)

### 4. 시각적 리듬 (Visual Rhythm)

반복적인 간격 패턴으로 시각적 일관성을 만듭니다.

- **균일 리듬:** 동일한 간격의 반복 (리스트, 그리드)
- **교대 리듬:** 두 가지 간격의 교대 (콘텐츠/구분선)
- **점진 리듬:** 점점 커지거나 작아지는 간격

```
균일:   8 - 8 - 8 - 8 - 8
교대:   8 - 16 - 8 - 16 - 8
점진:   8 - 12 - 16 - 24 - 32
```

### 5. 밀도 수준 (Density Levels)

용도에 따라 3가지 밀도 수준을 정의합니다:

| 밀도 | 요소 높이 | 아이템 간격 | 패딩 | 용도 |
|---|---|---|---|---|
| **Compact** | 32px | 4-8px | 4-8px | 데이터 테이블, 사이드바, 파워유저 UI |
| **Comfortable** | 40px | 8-16px | 8-16px | 일반 웹 UI, 대시보드 |
| **Spacious** | 48px | 16-24px | 16-24px | 모바일 UI, 마케팅 페이지, 접근성 우선 |

한 화면에서 밀도 수준은 최대 2가지까지 혼합할 수 있습니다.
예: 사이드바(Compact) + 메인 콘텐츠(Comfortable)


## 맥락별 적용

### Web

**반응형 간격 스케일:**

| 뷰포트 | 컬럼 수 | 거터 | 사이드 마진 | 기본 간격 |
|---|---|---|---|---|
| < 600px (Mobile) | 4 | 16px | 16px | 8px |
| 600-904px (Tablet) | 8 | 16px | 24px | 8px |
| 905-1240px (Laptop) | 12 | 24px | 32px | 8px |
| 1240-1440px (Desktop) | 12 | 24px | auto | 8px |
| > 1440px (Large) | 12 | 32px | auto | 8px |

- 최대 콘텐츠 너비: 1200-1440px
- 사이드 마진이 `auto`인 경우 콘텐츠를 중앙 정렬합니다.
- 모바일에서 데스크톱으로 전환 시 간격을 비례적으로 증가시킵니다.

**일반적인 웹 레이아웃 패턴:**
- 네비게이션 바 높이: 56-64px
- 사이드바 너비: 240-280px (닫힘: 56-72px)
- 카드 간격: 16-24px
- 폼 필드 간격: 16-24px
- 페이지 섹션 간격: 48-96px

### Mobile App

- 터치 친화적 간격: 최소 8px 아이템 간 여백
- 엄지 영역(thumb zone) 고려한 배치
- 세이프 에어리어(safe area) 내 콘텐츠 배치
- 상태 바, 네비게이션 바 높이 고려

**iOS 기본 레이아웃 값:**
- 사이드 마진: 16px (iPhone), 20px (iPad)
- 리스트 아이템 높이: 44pt
- 네비게이션 바: 44pt (Large title: 96pt)
- 탭 바: 49pt
- 셀 간격: 0 (구분선 사용) 또는 8pt

**Android 기본 레이아웃 값:**
- 사이드 마진: 16dp
- 리스트 아이템 높이: 48-72dp
- 앱 바: 56dp (세로) / 48dp (가로)
- 하단 네비게이션: 56dp
- 아이템 간격: 8-16dp

### Print

- 마진: A4 기준 최소 20mm (상하좌우)
- 거터: 양면 인쇄 시 제본 여백 10-15mm 추가
- 컬럼 간격: 5-10mm
- 단락 간격: 줄 높이의 0.5-1배
- 이미지와 텍스트 간격: 3-5mm
- 블리드(bleed): 재단 여백 3mm

### Presentation

- 슬라이드 마진: 전체 크기의 5-10%
- 제목과 본문 간격: 40-60px
- 불릿 아이템 간격: 24-40px
- 이미지와 텍스트 간격: 24-40px
- 하단 여백 확보: 프로젝터 하단 클리핑 고려


## ODE 매핑

ODE 포맷에서 공간 구성 관련 속성이 어떻게 표현되는지 설명합니다.

### LayoutConfig: 자동 레이아웃

프레임의 `container.layout` 속성으로 자동 레이아웃을 설정합니다.

```json
{
  "stable_id": "card-01",
  "name": "Card",
  "kind": {
    "type": "frame",
    "data": {
      "width": 320.0,
      "height": 0.0,
      "height_sizing": "Hug",
      "container": {
        "children": ["card-image", "card-content"],
        "layout": {
          "direction": "Vertical",
          "primary_axis_align": "Min",
          "counter_axis_align": "Min",
          "padding": {
            "top": 0.0,
            "right": 0.0,
            "bottom": 0.0,
            "left": 0.0
          },
          "item_spacing": 0.0,
          "wrap": "NoWrap"
        }
      }
    }
  }
}
```

### Padding: 내부 여백

`layout.padding`으로 컨테이너 내부 여백을 설정합니다.

```json
{
  "layout": {
    "direction": "Vertical",
    "padding": {
      "top": 24.0,
      "right": 24.0,
      "bottom": 24.0,
      "left": 24.0
    },
    "item_spacing": 16.0
  }
}
```

**패딩 권장값:**
- 카드 내부: 16-24px (4x-6x)
- 버튼 내부: 수직 8-12px, 수평 16-24px
- 입력 필드: 수직 8-12px, 수평 12-16px
- 페이지 컨테이너: 16-64px (반응형)

### item_spacing: 아이템 간 간격

`layout.item_spacing`으로 자식 요소 간 간격을 설정합니다.

```json
{
  "layout": {
    "direction": "Vertical",
    "item_spacing": 16.0
  }
}
```

**간격 권장값:**
- 인라인 요소 (아이콘+텍스트): 4-8px
- 리스트 아이템: 8-12px
- 폼 필드: 16-24px
- 카드: 16-24px
- 섹션: 32-64px

`spacing_scale` 체커는 item_spacing 값이 기본 그리드(4px 또는 8px)의
배수인지 확인합니다. tolerance 파라미터로 허용 오차를 설정합니다.

### Transform: 절대 위치

자동 레이아웃 외에 절대 위치가 필요한 경우 transform을 사용합니다.

```json
{
  "stable_id": "badge-01",
  "name": "Notification Badge",
  "transform": {
    "a": 1.0, "b": 0.0,
    "c": 0.0, "d": 1.0,
    "tx": 280.0, "ty": 8.0
  }
}
```

`transform.tx`와 `transform.ty`가 위치(position)에 해당합니다.
절대 위치 요소도 그리드에 정렬하는 것이 좋습니다:
- tx, ty 값을 4px 또는 8px 배수로 설정

### 밀도 수준 표현

밀도 수준에 따른 프레임 설정 예시:

**Compact (데이터 테이블 행):**

```json
{
  "kind": {
    "type": "frame",
    "data": {
      "width": 0.0,
      "width_sizing": "Fill",
      "height": 32.0,
      "container": {
        "layout": {
          "direction": "Horizontal",
          "padding": { "top": 4.0, "right": 16.0, "bottom": 4.0, "left": 16.0 },
          "item_spacing": 8.0
        }
      }
    }
  }
}
```

**Comfortable (일반 리스트 아이템):**

```json
{
  "kind": {
    "type": "frame",
    "data": {
      "width": 0.0,
      "width_sizing": "Fill",
      "height": 48.0,
      "container": {
        "layout": {
          "direction": "Horizontal",
          "padding": { "top": 12.0, "right": 16.0, "bottom": 12.0, "left": 16.0 },
          "item_spacing": 12.0
        }
      }
    }
  }
}
```

**Spacious (큰 카드 아이템):**

```json
{
  "kind": {
    "type": "frame",
    "data": {
      "width": 0.0,
      "width_sizing": "Fill",
      "height": 64.0,
      "container": {
        "layout": {
          "direction": "Horizontal",
          "padding": { "top": 16.0, "right": 24.0, "bottom": 16.0, "left": 24.0 },
          "item_spacing": 16.0
        }
      }
    }
  }
}
```

### 그리드 레이아웃 (12 컬럼 그리드)

ODE에서 그리드 기반 레이아웃을 구성하는 방법:

```json
{
  "stable_id": "page-01",
  "name": "Page Container",
  "kind": {
    "type": "frame",
    "data": {
      "width": 1200.0,
      "height": 0.0,
      "height_sizing": "Hug",
      "container": {
        "children": ["sidebar-01", "main-content-01"],
        "layout": {
          "direction": "Horizontal",
          "padding": { "top": 0.0, "right": 0.0, "bottom": 0.0, "left": 0.0 },
          "item_spacing": 24.0
        }
      }
    }
  }
}
```

사이드바(3 컬럼, 280px)와 메인 콘텐츠(9 컬럼) 간의
간격이 24px(3x 8px grid)로 설정된 예시입니다.


## 안티패턴

디자인 에이전트가 자주 범하는 공간 구성 관련 실수 목록입니다.

### 1. 균일한 간격 남용

**문제:** 모든 요소 사이에 동일한 간격(예: 16px)을 사용하여
시각적 그룹핑이 사라짐.

**해결:** 근접성 원리를 적용합니다. 관련 요소는 좁은 간격,
비관련 요소는 넓은 간격을 사용합니다.

```
나쁜 예: heading -16px- text -16px- divider -16px- heading -16px- text
좋은 예: heading -8px- text -32px- divider -32px- heading -8px- text
```

### 2. 그리드 무시

**문제:** 간격 값이 4px/8px 배수가 아닌 임의의 값 (예: 13px, 27px, 55px).

**해결:** 모든 간격을 4px grid에 맞춥니다.
- 4, 8, 12, 16, 20, 24, 32, 40, 48, 64

### 3. 과도한 밀도

**문제:** 가능한 많은 정보를 화면에 표시하려고
요소를 너무 밀접하게 배치.

**해결:** 밀도 수준을 의식적으로 선택합니다.
- 모바일: Spacious 밀도 기본
- 웹 대시보드: Comfortable 밀도 기본
- 데이터 테이블: Compact 밀도 허용

### 4. 여백 부족

**문제:** 페이지 가장자리까지 콘텐츠가 꽉 차 있어서
답답한 느낌. 사이드 마진이 0 또는 4px.

**해결:** 적절한 마진을 확보합니다.
- 모바일: 최소 16px 사이드 마진
- 데스크톱: 최소 24px, 권장 32-64px 사이드 마진

### 5. 일관성 없는 정렬

**문제:** 같은 페이지 내에서 좌측 정렬, 중앙 정렬, 우측 정렬이
무질서하게 혼합.

**해결:** 주요 정렬 축을 결정하고 일관되게 유지합니다.
- 본문 콘텐츠: 좌측 정렬 (LTR)
- CTA, 히어로: 중앙 정렬 허용
- 숫자 데이터: 우측 정렬

### 6. 패딩 불균형

**문제:** 컨테이너 내부의 상하좌우 패딩이 불균형.
예: top: 24px, right: 16px, bottom: 8px, left: 16px.

**해결:** 의도적인 비대칭이 아닌 한 대칭 패딩을 사용합니다.
- 균일: padding 24px (모든 방향)
- 비대칭: 수직 16px, 수평 24px (의도적)

### 7. 중첩 컨테이너의 이중 패딩

**문제:** 외부 컨테이너(padding: 24px) 안에 내부 컨테이너
(padding: 24px)가 있어서 실제 간격이 48px이 됨.

**해결:** 중첩 레벨에 따라 패딩을 조절합니다.
외부 컨테이너에 패딩이 있으면 직접 자식은
패딩을 줄이거나 제거합니다.

### 8. 반응형 미고려

**문제:** 데스크톱 레이아웃의 간격을 모바일에서도
동일하게 적용하여 콘텐츠가 잘리거나 너무 좁아짐.

**해결:** 뷰포트에 따라 간격 스케일을 조정합니다.
ODE의 constraints 시스템과 layout sizing을 활용하여
반응형 간격을 구현합니다.

### 9. 마이크로/매크로 혼동

**문제:** 아이콘과 텍스트 사이에 32px, 섹션 간에 8px처럼
마이크로/매크로 여백이 뒤바뀜.

**해결:** 관계의 강도에 따라 간격을 비례시킵니다.
- 강한 관계 (아이콘+레이블): 4-8px
- 중간 관계 (같은 그룹 내 요소): 8-16px
- 약한 관계 (다른 그룹): 16-32px
- 분리 (섹션 경계): 32-64px
